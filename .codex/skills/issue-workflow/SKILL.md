# Skill: Issue Workflow

## Goal
Run a consistent, low-friction issue lifecycle for active/project-tracking files.

## Steps
1. Read `ISSUES.md` and select next `unstarted`/`reopened` issue.
2. Set selected item to `in-progress`.
3. Implement scoped changes.
4. Verify with appropriate tests/build checks.
5. Update issue to `complete` with `Resolution:` note.
6. Move completed entry to `ISSUES_ARCHIVE.md`.
7. Keep `ISSUES.md` focused on active work only.

## Constraints
- Do not start reopened issues unless explicitly asked.
- Do not keep completed items in active list.
- Preserve previous `Resolution:` history when re-archiving reopened issues.
